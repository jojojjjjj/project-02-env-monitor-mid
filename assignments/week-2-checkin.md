## ✅ 一、本周进度检查清单 | Progress Checklist

> 对照 `curriculum/day-06.md` ~ `day-10.md` 的检查点逐项确认。

### Day 6 · WiFi 联网 + SNTP 网络授时 (WiFi + SNTP)

- [ ] 理解 SNTP/NTP 的作用：从网络时间服务器获取**准时间**，避免掉电后时间错乱
- [ ] 用 AT 命令让 ESP8266 连上 WiFi：
  - [ ] `AT+CWMODE=1` 设为 Station 模式
  - [ ] `AT+CWJAP="SSID","PASSWORD"` 连上你的 WiFi（收到 `WIFI CONNECTED` + `WIFI GOT IP`）
  - [ ] `AT+CIFSR` 拿到 IP 地址
- [ ] 用 AT 命令做 SNTP 授时：
  - [ ] `AT+CIPSNTPCFG=1,8,"ntp1.aliyun.com"` 配置时区（8 = UTC+8 北京时间）和 NTP 服务器
  - [ ] `AT+CIPSNTPTIME?` 查询时间，拿到形如 `Thu Jan 01 12:00:00 2026` 的时间字符串
- [ ] 在 STM32 端解析这个时间字符串，拆出**年月日时分秒**
- [ ] 把解析出的时间存入本地时间变量（`clock.c` 里的时间结构）
- [ ] 串口打印出同步到的时间，与手机时间核对（误差 < 2 秒）

### Day 7 · 时钟整合 — 时间 + 温湿度显示 (Integration)

- [ ] 调用新增的 `OLED_ShowTime(h, m, s)` 在 OLED 上显示 `HH:MM:SS`
- [ ] OLED 分区显示：上方时间、下方温湿度（或左右分栏，布局清晰）
- [ ] 时间每秒更新（用 SysTick 或 HAL_Delay 节拍）
- [ ] 温湿度每 5–10 秒刷新一次（不用每秒读，省功耗也避免 SHT31 自热）
- [ ] 时间走时准确：对照手机，1 分钟内不偏（本地计时靠 SysTick tick）
- [ ] 演示：上电 → 联网 → 授时 → OLED 显示正确时间 + 温湿度，全流程顺畅

### Day 8 · 完善 — 整点报时 / 断网回退 (Polish)

- [ ] **整点报时 (Hourly Chime):** 每到 `MM==00 && SS==00`，蜂鸣器（TIM2 PWM）响一声短鸣
- [ ] （可选）按键调时：按一个 GPIO 按键能手动 +1 分钟（用于无 WiFi 场景）
- [ ] **断网回退 (Offline Fallback):** 拔掉 ESP8266 或关掉 WiFi 路由器，时钟**继续走**（用本地 SysTick 计时，不卡死）
- [ ] **重连恢复 (Reconnect):** WiFi 恢复后，能重新做一次 SNTP 校时（不需要重启）
- [ ] 异常处理：SHT31 读失败时 OLED 显示 `T:--`，不崩机
- [ ] 连续运行 ≥10 分钟无崩溃、无死机

### Day 9 · 组装 — 焊接 / 外壳 (Assembly)

- [ ] （面包板版 MVP）面包板接线整理整齐，用扎带/胶带固定，能整体移动不松脱
- [ ] （或 PCB 版进阶）PCB 已打样到货，元件已焊接，通电测试正常
- [ ] 做了简单外壳（纸盒 / 3D 打印 / 亚克力），OLED 露出，整洁成桌面摆件
- [ ] 供电方式确认：USB 供电稳定（或加电池方案，可选）
- [ ] 整机外观拍照存档（用于最终展示幻灯片）

### Day 10 · 调试 + 最终展示准备 (Final Prep)

- [ ] 整机功能全流程跑通 ≥3 次，确认稳定
- [ ] 整理一份演示用串口日志（保存 `demo_log.txt`，含联网、授时、温湿度行）
- [ ] 准备好最终展示幻灯片（见 `final-presentation.md` 大纲）
